Networking Module - 7 Days

Day 1: Introduction to networking

We began with the very building blocks of networking—how information travels.

Key components: sender, receiver, transmission medium (cables, wireless), message & protocol.

Modes of communication: simplex, half-duplex, full-duplex with practical examples like walkie-talkies vs. video calls.

Analog vs digital signals: why digital dominates in modern systems (noise immunity, easier processing).

Essential characteristics: speed, accuracy, reliability, security.

Real-world link: how a single WhatsApp message travels as packets from one phone to another through multiple networks.

Day 2: Types of Network & Nature of Data

We zoomed out to see how networks scale and how different types of data flow through them.

LAN, MAN, WAN, PAN: From a home Wi-Fi network to global ISPs.
Client-server vs peer-to-peer models and their impact on scalability.
Bandwidth & latency trade-offs—why high-frequency trading needs ultra-low latency links.

Kinds of data: oriented (structured), observed (sensor data), inferred (predictions/analytics).
Glimpse into IoT ecosystems where these network types coexist..

Day 3: The OSI Model Demystified

The famous 7-layer OSI stack stopped being a buzzword and became a roadmap.

Layers: Physical, Data Link, Network, Transport, Session, Presentation, Application.

Protocol examples: Ethernet, IP, TCP/UDP, SSL/TLS, HTTP/HTTPS, FTP.

Encapsulation & de-encapsulation: how data is wrapped with headers and footers at each layer.

Hands-on: tracing a packet’s journey from a laptop to a website using ping and traceroute.

Day 4: Inside the Web: HTTP & DNS

We opened the black box of the browser.

DNS as the internet’s phonebook: converting example.com to an IP address.
HTTP/HTTPS request methods—GET, POST, PUT, DELETE—and how RESTful APIs use them.
Caching & response codes (2xx, 3xx, 4xx, 5xx) to handle web traffic efficiently.

Lab: using nslookup & browser dev-tools to watch a real DNS lookup in action.

Day 5: Revision & DHCP in Action

A quick OSI recap led to automatic IP management with DHCP.

Difference between static and dynamic IPs.
DHCP handshake: Discover → Offer → Request → Acknowledge.
Checked a live router’s DHCP lease table and learned how ISPs dynamically assign addresses.

Day 6: Low-Level Protocols & Diagnostics

We explored the protocols that quietly keep networks alive.

ARP (Address Resolution Protocol): maps IP to MAC addresses in a LAN.

ICMP (Internet Control Message Protocol): the backbone of tools like ping & traceroute.

DHCP Acknowledge: final confirmation of IP allocation.

Practical: tracing packets and identifying network bottlenecks using these tools.

Day 7: From Theory to Zynq-Based App

All the learning converged into a mini-project.

Designed & submitted a Zynq FPGA-based application, applying concepts like DHCP, ARP & ICMP.
Understood how embedded systems communicate with larger networks.
Learned how these skills transfer to IoT devices, industrial automation, and edge computing.
